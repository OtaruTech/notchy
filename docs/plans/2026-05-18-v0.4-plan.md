## Phase 1 — HUDFeature + Volume HUD (proof of concept)

### 1.1 Data shape

```swift
struct HUDEvent: Equatable, Sendable {
    enum Kind: Sendable { case volume, brightness, keyboardBacklight, mute }
    let kind: Kind
    let level: Double  // 0.0 - 1.0
    let muted: Bool
}
```

### 1.2 HUDFeature

```swift
@MainActor @Observable
final class HUDFeature {
    private(set) var current: HUDEvent?
    private var dismissTask: Task<Void, Never>?

    func show(_ event: HUDEvent, duration: TimeInterval) {
        current = event
        dismissTask?.cancel()
        dismissTask = Task { [weak self] in
            try? await Task.sleep(for: .seconds(duration))
            await MainActor.run {
                if self?.current == event { self?.current = nil }
            }
        }
    }
}
```

### 1.3 VolumeMonitor

`AudioObjectAddPropertyListenerBlock` on default output device for `kAudioHardwareServiceDeviceProperty_VirtualMainVolume` + `kAudioDevicePropertyMute`. On change → call `HUDFeature.show(volume:)`.

### 1.4 HUDView

Reuses `LiveActivityStrip`'s right wing or own slot. Shows speaker icon + filled bar + percent.

### 1.5 Wiring

- AppDelegate creates `HUDFeature` + `VolumeMonitor`
- NotchExpandedView: in collapsed states, if `hudFeature.current != nil`, render `HUDView` instead of (or layered on top of) the live-activity strip's right wing
- Settings: `notchy.hudVolumeEnabled` toggle

**DoD:** Press F10/F11/F12 → HUD shows for 1.5s, dismisses cleanly.

**Commit:** `feat(hud): volume HUD takeover via CoreAudio listener`

## Phase 2 — Brightness + Keyboard backlight HUDs

Same pattern, two more monitors:

### 2.1 BrightnessMonitor

`CGDisplayRegisterReconfigurationCallback` + `IOServiceMatching("AppleBacklightDisplay")` → poll the `display_brightness` IOService property.

Fallback when private IOService access fails: hook F1/F2 via Carbon hotkey + read brightness via `display.brightness` AppleScript bridge.

### 2.2 KeyboardBacklightMonitor

`IOServiceMatching("AppleHIDKeyboardEventDriverV2")` → keyboard backlight level property. Same private-API caveats; same fallback.

### 2.3 Wiring

`HUDEvent.Kind.brightness` + `.keyboardBacklight` already in enum. `HUDView` renders different icon per kind. Settings toggles for each.

**DoD:** F1/F2 → brightness HUD; F5/F6 → keyboard HUD.

**Commit:** `feat(hud): brightness + keyboard backlight HUD`

## Phase 6 — Caffeine

### 6.1 CaffeineController

```swift
@MainActor
final class CaffeineController {
    private var process: Process?

    func toggle() {
        if process != nil { stop() } else { start() }
    }

    private func start() {
        let p = Process()
        p.launchPath = "/usr/bin/caffeinate"
        p.arguments = ["-d", "-i", "-m"]
        try? p.run()
        process = p
    }

    private func stop() {
        process?.terminate()
        process = nil
    }
}
```

### 6.2 Wiring

- Dashboard row: "Keep awake [○]" toggle switch
- HotKeyCenter: new `.toggleCaffeine` action (⌘⌥K)
- Settings: `notchy.indicatorCaffeineEnabled`

**DoD:** Toggle → `caffeinate` running in `ps aux`. Toggle again → killed.

**Commit:** `feat(status): caffeine toggle + ⌘⌥K hotkey`
