# Notchy v0.6 — Daily polish

**Date:** 2026-05-18
**Status:** Draft → implementation
**Scope:** Three independent, daily-use features. Smaller than v0.5.

---

## Tracks

### A. 🔔 Lark / 飞书 unread badge

**Goal:** Show the count of unread Lark messages in the dashboard so the
user doesn't need to look at the Dock.

**Approach:** Read the Dock process via Accessibility API.
- Find the Dock process by bundle ID `com.apple.dock`
- Walk `AXChildren` of its first list (the application icons list)
- For each tile, read `AXTitle` (app name) and `AXValue` (badge string, e.g. "12")
- Match `AXTitle == "Lark" || "飞书"`
- Push count into `SystemStatusFeature.larkUnread`

**Cadence:** 5s poll. Cheap because Dock's AX tree is small.

**UI:** Dashboard row `🔔 飞书 12` (chrome-blue tint), clickable → activates Lark.app. Hidden when `0` or Lark not running.

**Gated by:** `notchy.indicatorLarkEnabled` (default `true`).

### B. 🍅 Pomodoro stats + heatmap

**Goal:** "How many focus sessions did I do today?" answered at a glance.

**Persistence:** Append-only log file at
`~/Library/Application Support/tech.otaru.Notchy/pomodoro-log.json`.
Each row: `{ "completedAt": 1735000000, "durationMin": 25 }`.

**Stats computed:**
- Today: count + total focus minutes
- Yesterday: count
- This week: count
- Streak: consecutive days with ≥1 session

**UI changes:**
1. **TimerView** — small stats footer when state is `.idle`:
   ```
   Today: 4 · This week: 17 · Streak: 3 days
   [▪ ▪ ▪ ▪ ▫ ▫ ▪]  ← 7-day dots, filled = ≥1 session
   ```
2. **Dashboard row** — `🍅 4 today · 100m` (only when count > 0)

**Hookup:** `TimerFeature.tick()` → on `nr <= 0` (timer completed) → append log entry. Cap log at 1000 entries.

### C. ⌨️ Customisable hotkeys

**Goal:** Let users remap the 4 global hotkeys without recompiling.

**Default bindings:**
- ⌘⌥N — toggle dashboard
- ⌘⌥M — toggle Mirror
- ⌘⇧V — clipboard panel
- ⌘⌥K — Caffeine toggle

**Recording UI:** Settings → General → Keyboard shortcuts section gets a
recorder row per action:
```
Toggle dashboard:  [ ⌘⌥N ]  (click to record)
```
Click → field shows "Press a key…" → next key chord captured → store.

**Persistence:** UserDefaults dictionary `notchy.hotkeys` with action → `{keyCode, modifiers}`.

**HotKeyCenter changes:** Read bindings from UserDefaults on `start()`. Each registered hotkey stores a callback per action. New method `reloadBindings()` re-registers from current UserDefaults — Settings calls this on every change so the user sees instant effect without restart.

**Encoding:** `{keyCode: Int, modifierFlags: UInt32}` where modifier flags are Carbon's `cmdKey | optionKey | controlKey | shiftKey` OR'd.

**Conflict prevention:** Reject Cmd-Q (terminate), Cmd-Tab, plain F-row keys (already used by HUD). Show inline warning if user tries.

## Non-goals

- Auto-detect Claude Code / Cursor AI activity (no reliable signal)
- Pomodoro categories ("focus on X" tags) — UI scope creep
- Multi-display panel awareness — defer
- Clipboard pinboards / stack mode — defer to v0.7
