## 1 · Background & motivation

Clipboard managers are one of the highest-frequency productivity utilities on macOS. Power users hit ⌘C / ⌘V dozens of times an hour and routinely lose context — overwriting a useful copy, switching apps and forgetting what was on the clipboard, copying the same colour value four times in a Figma session.

Paste ([pasteapp.io](https://pasteapp.io)) is the gold-standard commercial entry — 60k+ paying users, $30/yr, signature floating bar. Maccy and Raycast both ship strong free clipboards. None of them have a notch presence: they steal screen real estate when activated.

**Notchy's wedge:** the clipboard panel is *anchored on the notch*. ⌘⇧V drops the panel out from under the notch as a wide editorial strip. Items live where the eye is already trained.

This PRD scopes the v0.3 → v0.4 clipboard work.

## 3 · Personas

**P1. Dev / IC** — Switches between IDE, terminal, Slack, browser. Constantly copies stack traces, file paths, URLs, snippets. Average **150–300 clipboard events / day**. Cares about: search, code formatting, plain-text paste.

**P2. Designer** — Lives in Figma. Copies hex colours, asset URLs, design specs to Slack. **80–120 / day**. Cares about: colour swatches, image thumbnails, ability to scrub recent copies fast.

**P3. PM / writer** — Drafts notes, copies excerpts, builds reports. **40–80 / day**. Cares about: search, retaining formatting, browser links with rich preview.

**P4. Researcher / student** — Highlights from PDFs and webpages while writing. Long text copies. Cares about: history depth, text search.

Common: nobody wants the clipboard panel to feel slow, ugly, or in the way.

## 5 · Functional requirements

### 5.1 Capture

| # | Requirement |
|---|-------------|
| C1 | Poll `NSPasteboard.general.changeCount` every 500 ms; on change, snapshot the active item |
| C2 | Skip captures from excluded bundle IDs (default: 1Password, Bitwarden, Keychain Access, Dashlane, LastPass) |
| C3 | Honour the `org.nspasteboard.ConcealedType` / `org.nspasteboard.AutoGeneratedType` UTI hints — skip if present |
| C4 | Deduplicate: if the new item's content hash matches the most recent item, bump its `updatedAt` instead of inserting a duplicate row |
| C5 | Detect item kind in priority order: image → file URL → colour hex/rgb → URL → code → rich text → plain text |
| C6 | Cap stored items per kind: 1000 text/url/colour, 500 code, 200 image, 200 file. LRU eviction. |
| C7 | Persist source app (bundle id + display name + icon) per item |
| C8 | Pause-capture toggle (hotkey ⌘⇧P — `notchy.clipboardPaused`) |
| C9 | Auto-purge items older than N days (default 30, settings 0 / 7 / 30 / 90 / never) |

### 5.2 Recall (the notch panel)

| # | Requirement |
|---|-------------|
| R1 | Hotkey **⌘⇧V** opens the clipboard panel; same hotkey toggles it closed |
| R2 | Panel emerges from under the notch with a 0.25s spring expansion; height ≈ 280pt, width 800pt |
| R3 | Top row: search field (auto-focused) + small "(N items)" count + pause indicator if paused |
| R4 | Body: horizontal scroll of cards. Each card 120pt wide × 160pt tall |
| R5 | Each card shows, top-to-bottom: type icon + source-app icon (small), preview content, 1-line meta (`12s ago · Figma`) |
| R6 | **Keyboard navigation**: arrow keys move selection; **1-9** pastes that item directly; **Enter** pastes selected; **Esc / ⌘⇧V** closes |
| R7 | **Click** an item → pastes & dismisses |
| R8 | **⌘C on a selected item** → copies back to clipboard without pasting (panel stays open) |
| R9 | **⌫ / Del** on a selected item → soft-delete with undo toast |
| R10 | Type-to-search: as the user types in the search field, items filter live by content + source-app match |
| R11 | When the panel pastes an item, the user's **previous clipboard content is restored** ~50 ms after the paste (configurable; on by default) |
| R12 | If no items match, render an empty state with hint (`⌘V somewhere first…`) |

### 5.3 Item kinds & previews

| Kind | Detection | Preview UI |
|------|-----------|------------|
| **Image** | pasteboard contains `public.image` | Thumbnail, `WxH` overlay, file size |
| **File** | pasteboard contains `public.file-url` | File icon (Finder API), filename, parent folder name |
| **Color** | string matches `^#?[0-9a-fA-F]{3,8}$` or `rgb(.+)` / `rgba(.+)` | Coloured swatch fills card top, hex value below |
| **URL** | string matches strict URL regex with http(s) scheme | Favicon (cached), URL, page title if cached |
| **Code** | string contains ≥ 3 lines AND ≥ 70% non-prose characters | Monospaced excerpt, language guess in corner |
| **Rich text** | pasteboard contains `public.rtf` AND text > 0 | Truncated rendered preview |
| **Plain text** | fallback | Truncated text, max 4 lines |

### 5.4 Paste mechanics

| # | Requirement |
|---|-------------|
| P1 | Paste = write item to pasteboard, then synthesise ⌘V via `CGEvent` to the *previously-focused* application |
| P2 | Track the previously-focused app via `NSWorkspace.shared.frontmostApplication` snapshot **before** the panel opens |
| P3 | After paste fires, restore the user's prior clipboard content after a 50 ms delay (only when `notchy.clipboardRestore = true`, default true) |
| P4 | If the synthesised ⌘V fails (no Accessibility permission), fall back to leaving the item on the clipboard and showing a toast: "Press ⌘V to paste" |

### 5.5 Settings (Settings → Clipboard tab)

| Key | Default | Description |
|-----|---------|-------------|
| `notchy.clipboardEnabled` | true | Master on/off |
| `notchy.clipboardPaused` | false | Temporarily pause capture |
| `notchy.clipboardMaxItems` | 1000 | Cap for text/url/colour |
| `notchy.clipboardRetentionDays` | 30 | Auto-delete after N days; 0 = never |
| `notchy.clipboardRestore` | true | Restore previous clip after paste |
| `notchy.clipboardCaptureImages` | true | Toggle image capture (can be heavy) |
| `notchy.clipboardExcludedBundleIDs` | `[com.1password.macos, com.agilebits.onepassword*, com.bitwarden.desktop, com.apple.keychainaccess]` | Comma-separated bundle IDs to skip |
| `notchy.clipboardHotkey` | ⌘⇧V | Configurable later |

### 5.6 Security & privacy

| # | Requirement |
|---|-------------|
| S1 | All data lives in `~/Library/Application Support/tech.otaru.Notchy/clipboard.sqlite` |
| S2 | DB file mode 0600 (user-only) |
| S3 | Images stored as separate files in `…/clipboard/images/<uuid>.png`, referenced by row |
| S4 | NEVER send any clipboard data over the network. No analytics. |
| S5 | "Clear all" button in Settings + hotkey ⌘⇧⌫ (configurable) |
| S6 | "Forget last item" hotkey (placeholder for v0.4) |
| S7 | Respect macOS `ConcealedType` pasteboard UTI as a hard-skip |

## 6 · UX flows

### 6.1 First copy → first paste from history (cold path)

1. User copies `git rebase -i HEAD~3` from terminal
2. Half a second later, user copies a URL from browser
3. User cmd-tabs into Slack, hits **⌘⇧V**
4. Notch expands downward in 0.25 s, search field is focused
5. Two cards visible: URL (top-of-list), code snippet (next)
6. User hits **2** → URL is pasted into Slack; previous clipboard (the URL) is restored 50 ms later

Total elapsed: ~1.5 s from hotkey to paste landing.

### 6.2 Search and pin (warm path)

1. User has 200+ items, needs the colour they copied 30 min ago
2. ⌘⇧V → types `#3a` → list filters to colour items starting with `#3a…`
3. Arrow-right to scroll, Enter to paste

### 6.3 Excluded app

1. User unlocks 1Password, copies a password
2. Notchy notices change but reads frontmost = `com.1password.macos` → skips capture
3. Password never enters Notchy DB

### 6.4 Paused

1. User toggles "Pause capture" in menu bar (next to Settings)
2. Notchy continues to poll changeCount but ignores all changes
3. Menu bar icon dims to indicate paused state

- Number badge (1-9) in card top-right corner
- Selection ring: 2pt accent stroke

### 7.3 Animations

- Open: spring(response: 0.25, damping: 0.85), panel slides down
- Card hover: lift + 2pt shadow, 0.12 s ease
- Selection move: 0.18 s easeOut on selection ring
- Paste: card flashes accent for 80 ms, then panel fades out (0.18 s) and the previously-focused app receives ⌘V
- Delete: card collapses width to 0 in 0.20 s, neighbouring cards reflow

## 8 · Data model

```sqlite
CREATE TABLE items (
    id              TEXT PRIMARY KEY,          -- UUID
    kind            TEXT NOT NULL,             -- text|richtext|url|image|file|color|code
    content_hash    TEXT NOT NULL,             -- SHA-256 of payload; used for dedupe
    payload_text    TEXT,                      -- text/url/color/code/richtext (RTF)
    payload_path    TEXT,                      -- image file / file URL
    preview         TEXT,                      -- truncated string for list view
    source_bundle   TEXT,                      -- e.g. com.apple.Safari
    source_name     TEXT,                      -- "Safari"
    byte_size       INTEGER,                   -- approximate
    created_at      INTEGER NOT NULL,          -- unix epoch sec
    updated_at      INTEGER NOT NULL,
    pinned          INTEGER NOT NULL DEFAULT 0,
    deleted_at      INTEGER                    -- soft-delete
);

CREATE INDEX idx_items_updated ON items (updated_at DESC) WHERE deleted_at IS NULL;
CREATE INDEX idx_items_hash    ON items (content_hash);
CREATE INDEX idx_items_kind    ON items (kind, updated_at DESC) WHERE deleted_at IS NULL;
```

GRDB.swift (or plain `SQLite3` C API to avoid SwiftPM dependency creep).

## 11 · Risks & mitigations

| # | Risk | Mitigation |
|---|------|-----------|
| RK1 | NSPasteboard polling burns CPU | 500 ms interval; only read full content when `changeCount` actually changed |
| RK2 | Large image copies bloat DB | Off-load PNGs to separate files; cap image-history at 200; offer "Capture images: off" toggle |
| RK3 | Password manager leakage | Default-exclude known bundles + respect `ConcealedType` UTI; document the exclusion list |
| RK4 | Synthesised ⌘V fails in some apps (e.g., sandboxed) | Fall back to clipboard + toast; never lose the item |
| RK5 | Hotkey conflict with apps that use ⌘⇧V (e.g., VSCode paste-without-formatting) | Allow user to remap in Settings (v0.4) |
| RK6 | Panel feels slow on cold SQLite open | Lazy-open DB at first hotkey press; keep prepared statements |

## 13 · MVP scope (v0.3 ship target)

**In:**
- Capture (text, URL, image, file, colour, code, rich text)
- Persistence via SQLite (no external dep — use libsqlite3)
- ⌘⇧V notch panel with search, arrow nav, 1-9 paste, click paste, Enter paste, Esc close
- Paste with clipboard restoration
- App exclusions (default list + user-editable)
- Pause toggle in menu bar
- Settings tab: master on/off, retention, restore toggle, exclusion list, clear all
- Auto-purge of items older than `retentionDays`

**Out (deferred to v0.4):**
- Pinboards / named collections
- Stack mode
- Edit before paste
- Plain-text paste hotkey
- Smart actions (open URL, save image, etc.)
- Right-click context menu actions beyond delete

## 14 · Open questions

1. Use SwiftPM + GRDB.swift, or stay zero-dep with raw `libsqlite3`? **Decision: raw libsqlite3** — keeps build matrix simple and tracks Notchy's no-network ethos.
2. Should image previews show in cards immediately or only on hover? **Decision: immediately** — 200 max items, all in memory is fine.
3. Should we expose the database file location for user audit? **Decision: yes** — Settings → "Reveal data folder" button.
